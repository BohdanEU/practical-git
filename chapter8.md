# 8. Обґрунтування вибору типу бази даних

Для інформаційної системи інтернет-магазину обрано реляційну модель бази даних (SQL).  
Реляційні бази даних добре підходять для систем електронної комерції, оскільки вони
забезпечують високу узгодженість даних, підтримку транзакцій та чітку структуру
зв'язків між сутностями.

Основні сутності системи інтернет-магазину:

- Користувачі (Users)
- Товари (Products)
- Категорії товарів (Categories)
- Замовлення (Orders)
- Позиції замовлення (OrderItems)
- Оплати (Payments)

Між цими сутностями існують чітко визначені зв’язки:

- один користувач може мати багато замовлень;
- одне замовлення може містити багато товарів;
- товари належать до певних категорій.

Реляційна база даних дозволяє ефективно реалізувати ці зв'язки за допомогою первинних
та зовнішніх ключів, а також забезпечує цілісність даних.

Основними причинами вибору SQL-бази даних є:

1. **Транзакційність (ACID)**  
   При оформленні замовлення необхідно гарантувати, що всі операції (створення
   замовлення, списання товару зі складу, запис платежу) виконуються як одна
   транзакція.

2. **Чітка структура даних**  
   Дані інтернет-магазину мають чітку структуру та взаємозв'язки, що добре
   відповідає реляційній моделі.

3. **Підтримка складних запитів**  
   SQL дозволяє виконувати складні запити для аналітики, статистики продажів,
   формування звітів.

4. **Надійність та зрілість технології**  
   Реляційні СУБД широко використовуються у комерційних системах та мають
   стабільну екосистему.

У якості системи керування базами даних пропонується використовувати **PostgreSQL**.

Основні переваги PostgreSQL:

- підтримка складних SQL-запитів;
- висока надійність і стабільність;
- підтримка транзакцій;
- можливість масштабування;
- підтримка індексів, тригерів та процедур.

Таким чином, використання реляційної бази даних PostgreSQL є доцільним для
інформаційної системи інтернет-магазину, оскільки вона забезпечує надійне
зберігання даних, підтримку транзакцій і ефективну роботу з великими обсягами
інформації.
