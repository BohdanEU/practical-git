# 7. Побудувати модель даних

Для забезпечення роботи інформаційної системи інтернет-магазину необхідно розробити модель даних. Модель даних визначає основні сутності системи, їх атрибути та зв'язки між ними.

Основні сутності системи:

- Користувач (User)
- Товар (Product)
- Замовлення (Order)
- Кошик (Cart)
- Категорія товарів (Category)

## Сутність "Користувач"

Користувач представляє зареєстрованого клієнта системи.

Основні атрибути:

- id користувача
- ім'я
- електронна пошта
- пароль
- номер телефону

## Сутність "Товар"

Товар містить інформацію про продукцію, яка продається в інтернет-магазині.

Основні атрибути:

- id товару
- назва товару
- опис
- ціна
- кількість на складі

## Сутність "Замовлення"

Замовлення представляє покупку, здійснену користувачем.

Основні атрибути:

- id замовлення
- дата замовлення
- статус замовлення
- загальна сума

## Сутність "Категорія"

Категорія використовується для групування товарів.

Основні атрибути:

- id категорії
- назва категорії
- опис категорії

## Зв'язки між сутностями

Між сутностями існують такі зв'язки:

- один користувач може створити багато замовлень
- одне замовлення може містити багато товарів
- кожен товар належить до певної категорії
- користувач може додавати товари до кошика

Модель даних дозволяє структуровано зберігати інформацію та забезпечує ефективну роботу інформаційної системи.
