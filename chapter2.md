# 2. Провести збір і аналіз вимог

Для розробки інформаційної системи інтернет-магазину необхідно провести збір та аналіз вимог. Основною метою системи є автоматизація процесу продажу товарів через Інтернет, управління каталогом товарів, обробка замовлень та взаємодія з клієнтами.

## Аналіз предметної області

Інтернет-магазин являє собою інформаційну систему, яка забезпечує взаємодію між продавцем та покупцем через веб-інтерфейс. Основними процесами системи є перегляд каталогу товарів, оформлення замовлення, оплата товарів та управління замовленнями.

Основні бізнес-процеси системи:

- управління каталогом товарів
- реєстрація та авторизація користувачів
- оформлення замовлень
- обробка та підтвердження замовлень
- оплата товарів
- доставка товарів клієнтам

## Функціональні вимоги

Функціональні вимоги визначають основні можливості системи:

- система повинна дозволяти користувачам переглядати каталог товарів
- система повинна дозволяти користувачам шукати товари
- система повинна дозволяти додавати товари до кошика
- система повинна дозволяти оформлювати замовлення
- система повинна дозволяти адміністраторам додавати, редагувати та видаляти товари
- система повинна зберігати інформацію про замовлення

## Нефункціональні вимоги

Нефункціональні вимоги описують характеристики роботи системи:

- система повинна працювати цілодобово
- інтерфейс системи повинен бути зручним для користувачів
- система повинна забезпечувати захист персональних даних
- система повинна забезпечувати швидкий доступ до інформації

## Обмеження системи

Під час розробки системи необхідно врахувати такі обмеження:

- використання веб-технологій для доступу до системи
- використання бази даних для зберігання інформації
- обмеження доступу до адміністративних функцій системи

Результатом збору та аналізу вимог є формування специфікації вимог до інформаційної системи, яка буде використовуватися на наступних етапах проєктування.
