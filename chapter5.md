# 5. Відобразити User Stories у UML-нотації

Для кращого розуміння взаємодії користувачів із системою використовуються UML-діаграми. Одним із найбільш поширених типів UML-діаграм є діаграми послідовності (Sequence Diagram).

Діаграма послідовності показує порядок взаємодії між користувачем та системою під час виконання певного сценарію.

## Сценарій 1: Оформлення замовлення

У цьому сценарії клієнт обирає товар, додає його до кошика та оформлює замовлення.

```mermaid
sequenceDiagram
    participant Client
    participant Website
    participant Database

    Client->>Website: Перегляд каталогу товарів
    Website->>Database: Запит списку товарів
    Database-->>Website: Список товарів
    Website-->>Client: Відображення товарів

    Client->>Website: Додати товар до кошика
    Website->>Database: Зберегти товар у кошику

    Client->>Website: Оформити замовлення
    Website->>Database: Створити запис замовлення
    Database-->>Website: Підтвердження
    Website-->>Client: Замовлення оформлено
